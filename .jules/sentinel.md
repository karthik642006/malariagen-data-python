## 2025-02-27 - 🛡️ Sentinel: [CRITICAL] Fix Pandas Query Injection vulnerabilities
**Vulnerability:** Found critical SQL-injection style code execution vulnerability via Pandas `query()` method. Several files (`malariagen_data/util.py`, `anoph/genome_features.py`, `anoph/frq_base.py`, `anoph/karyotype.py`, `anoph/cnv_frq.py`) used f-strings to format user-provided variables directly into the query string (e.g., `df.query(f"contig == '{contig}'")`). Because pandas evaluates query expressions (often with `engine='python'`), an attacker providing a maliciously crafted string could break out of the quotes and execute arbitrary code.
**Learning:** Python's f-strings should never be used to construct strings evaluated by interpreters (Pandas query/eval, SQL engines, bash execution). When using `@` variable binding in `.query()` with keyword dictionaries like `**options`, prefer standard boolean indexing `df.loc[]` to avoid evaluation scope complexities.
**Prevention:** Always use Pandas variable binding `@var` syntax to safely bind local context to the query expression without inline evaluation, or use standard boolean indexing `df.loc[df["col"] == var]`.
