## 2024-05-24 - Fix pandas code injection vulnerabilities
**Vulnerability:** Pandas query evaluation code injection via Python f-strings. Several modules (e.g., `util.py`, `anoph/frq_base.py`, `anoph/genome_features.py`) used `df.query(f"var == '{val}'")` syntax which allows arbitrary code execution or logical bypass if variables contain malicious input, especially since `engine='python'` is often the default evaluator backend.
**Learning:** Python f-string interpolation into dynamic evaluation functions bypasses safe data bindings and leaves input unescaped. Using the built-in variable reference features of libraries (like Pandas' `@`) is required. In cases where the variable represents complex object properties (e.g., `r.start`), creating a local variable block (`rstart = r.start`) correctly exposes it to the evaluation scope without injecting raw code.
**Prevention:** Avoid `f"..."` strings entirely inside `.query()` or `.eval()`. Instead, rely on variable binding using `@` combined with safe local assignments. Add `# noqa: F841` to dummy assignments to avoid triggering strict linting rules.
