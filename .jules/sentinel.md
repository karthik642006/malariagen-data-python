## 2025-02-23 - [CRITICAL] Fix code injection in DataFrame query
**Vulnerability:** Found `pandas.DataFrame.query()` called with f-strings interpolating variables (e.g. `df.query(f"type == '{var}'")`). Since `engine="python"` is heavily used in this codebase to support extension array dtypes, f-string concatenation introduces severe code injection vulnerabilities.
**Learning:** `pandas.DataFrame.query()` and `.eval()` calls in this application commonly fall back to the Python engine. Developers incorrectly used Python's f-string formatting inside these evaluations, which could allow malicious payload execution within the evaluation namespace.
**Prevention:** Always use safe variable binding via `@` syntax (e.g. `df.query("type == @var")`) in pandas queries. The `@` operator resolves the variable directly rather than concatenating literal string values. Ensure any variables needed by the `@` operator are assigned to local variables and annotated with `# noqa: F841` to satisfy ruff when not explicitly "used" in normal python code execution.
