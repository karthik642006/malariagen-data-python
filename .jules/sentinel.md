## 2024-05-30 - pandas query and eval injection vulnerabilities
**Vulnerability:** Pandas `query()` and `eval()` were being used with f-strings to pass variables into the query strings (e.g., `df.query(f"contig == '{contig}'")`).
**Learning:** This is a high-priority code injection vulnerability. Since the codebase uses `engine='python'` to support extension types, arbitrary code execution is possible if the variable is user-controlled. Even if not user-controlled, string interpolation in pandas queries can cause crashes if variables contain quotes. Furthermore, variables containing dots like object attributes (`@r.end`) will raise errors in some Pandas execution engines.
**Prevention:** NEVER use f-strings or `.format()` to inject values into `pandas.DataFrame.query()` or `eval()`. Use local variable binding with the `@` prefix (`df.query("contig == @contig")`). If accessing an object attribute, assign it to a local variable first (e.g. `rend = r.end` followed by `df.query("start <= @rend")`) and append `# noqa: F841` to prevent linters from flagging it as unused.
