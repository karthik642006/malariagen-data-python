## 2024-05-30 - Fix code injection vulnerability in pandas.DataFrame.query()
**Vulnerability:** Code injection via string interpolation (f-strings) in `pandas.DataFrame.query()` expressions across multiple modules (`malariagen_data/util.py`, `malariagen_data/anoph/genome_features.py`, `malariagen_data/anoph/karyotype.py`, `malariagen_data/anoph/frq_base.py`, `tests/anoph/test_pca.py`).
**Learning:** The codebase defaults to `engine="python"` for pandas query evaluation to support extension array types. Using string interpolation inside these queries allows variables (like `contig` or `region` names) to break the query syntax or potentially execute arbitrary code if controlled by an attacker, effectively acting like a SQL injection vector against the pandas evaluation engine.
**Prevention:** Avoid string interpolation inside `.query()`. Instead, prefer using safe boolean indexing directly (e.g., `df.loc[df["col"] == val]`) to completely sidestep the string evaluation engine and any associated injection risks.
