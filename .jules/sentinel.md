## 2024-05-18 - Pandas query() string interpolation Code Injection Vulnerability
**Vulnerability:** String interpolation (f-strings) inside `pandas.DataFrame.query()` expressions (e.g., `df.query(f"contig == '{contig}'")`) introduces a severe code injection risk. Because `malariagen_data` uses `engine='python'` in some query evaluations to support extension types, unvalidated user input or parameters like `contig` or `self._gff_gene_type` can be injected directly into the evaluation scope and executed as arbitrary code.
**Learning:** Python-backed `pandas.DataFrame.query()` or `eval()` must never accept formatted strings constructed with unvalidated input. The evaluation engine executes the query as code, allowing injection attacks. This is exacerbated when pandas uses `engine="python"`.
**Prevention:** Always use safe standard boolean indexing (e.g., `df.loc[df["col"] == val]`) instead of `query()`, which bypasses the string evaluation engine entirely and natively prevents code execution from parameters.
