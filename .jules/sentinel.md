
## $(date +%Y-%m-%d) - [CRITICAL] Code Injection in Pandas DataFrame Query Evaluation
**Vulnerability:** A vulnerability was discovered in `malariagen_data/anoph/phenotypes.py` where `df_merged.query(sample_query, **(sample_query_options or {}))` was executing user-provided queries without restricting the evaluation environment. Because the `local_dict` and `global_dict` were not explicitly cleared and passed as empty, this allowed arbitrary code execution if a malicious payload was injected via the query options or query string in `pandas.DataFrame.query`.
**Learning:** This is an important security pattern specifically for projects relying on pandas. Pandas' `query()` and `eval()` use `eval()` under the hood which can run arbitrary code unless restricted. Although this had been addressed in other parts of the codebase (e.g. `base.py`, `sample_metadata.py`), `phenotypes.py` had an unprotected `query()` call.
**Prevention:** To prevent this, always extract and clear `local_dict` and `global_dict` from any kwargs before passing to `pandas.DataFrame.query()` or `eval()`. Use the `engine="python"` argument explicitly, and set `local_dict=local_dict` and `global_dict=global_dict` after retrieving them (defaulting to empty dictionaries `{}` to limit scope execution) while passing other options safely.
