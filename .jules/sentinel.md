## 2025-02-14 - Fix command injection in pandas query and eval
**Vulnerability:** pandas.DataFrame.eval() and .query() calls without explicit `local_dict` and `global_dict` restrictions allow arbitrary command injection if users supply queries containing `@` variable evaluation (e.g. `@os.system('...')`).
**Learning:** This repo makes extensive use of `pandas.DataFrame.eval()` and `query()` for user-supplied queries (`sample_query`, `variant_query`, `snp_query`). Some methods safely isolate the environment by setting `local_dict` and `global_dict` to `{}`, but others missed this protection, making them vulnerable. Furthermore, users should not use `@` references in queries targeting internal functions since they would inadvertently target internal variables.
**Prevention:** Always restrict `local_dict` and `global_dict` to `{}` or tightly controlled variables when passing user input directly to pandas evaluation methods, especially in library methods where the caller's scope isn't automatically protected by the evaluator.
